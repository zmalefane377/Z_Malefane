Process Documentation

1.Setting up Git:

  • Installed Git, created repositories, and practiced basic commands such as init, add, commit, push, and pull to understand version control.
  • Use the above link to download Git
  • When you have set up your Git account
  • Click on the code drop down arrow
  • Copy the url under the HTTPS
    1. Set up ssh key
    2. Open your terminal or command prompt.
    3. Generate a new SSH key: ssh-keygen -t ed25519 -C "your_email@example.com"
    4. Press Enter to accept the default file location.
    5. Secure passphrase when prompted
    6. Start the SSH agent: eval "$(ssh-agent -s)"
    7. Add your SSH private key to the agent: ssh-add ~/.ssh/id_ed25519
    8. Copy the public key to your clipboard
    9. Go to GitHub.com → Settings → SSH and GPG keys → New SSH key and Paste your public key and save.
  • Open your Git GUI
  • Command Git clone and paste the url
  • Make sure you have the corect repository as you may have created multiple in the first attempt
  • After cloning and forking, you should typically have a document in your pc where you can find all your .html

  • Key Words
    1. Git: A version control system used to track changes in code and collaborate with others.
    2. Repository (Repo): A storage location where project files and their history are kept.
    3. Commit: A saved change in a repository, like taking a snapshot of progress.
    4. Push/Pull: Sending (push) or retrieving (pull) code between a local repository and an online one.
    5. Debugging: The process of finding and fixing errors in code.
